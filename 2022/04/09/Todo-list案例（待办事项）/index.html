Todo-list案例（待办事项）

组件划分是按照功能点来划分的

起名字需要避开html标签名

一队数据用数组 每个数据的内容很多用对象保存(对象里面需要有id)

添加事件：先获取表单的值

1.键盘事件绑定@keyup.enter=”add”

methods:{

add(event){

console.log(event.target.value)获取事件的值

}}

2.双向数据绑定v-model

v-mode=’title’

data(){

return {

title: “

}}

methods:{

add(){

console.log(this.title)获取绑定的表单值

}}

将用户的输入包装成一个对象

两种情况：1.匹配服务器（只把title和done值交给服务器 服务器往据库里面写入数据 由数据库生成id）

2.单机版

子传父 （父亲先传给儿子一个函数 函数带有参数 儿子接收到函数 把需要传的数据以参数形式交给函数）

关于勾选 ：1.需要绑定事件 同时也必须拿到对应的id

2.绑定change事件（因为点击就改变了）

拿到id 通知对应项的done取反

数据在哪里 操作数据就在哪里

：checked=“todo.done” @change=’handleCheck(todo.id)’

等同于下面

v-model=‘todo.down’

不建议v-model 因为todo是通过props接收的对象 这个是只读的一般不可以修改 不报错是vue检测不到修改 vue检测到的是如下类似

obj.a修改vue检测不到 let obj={a:1，b:2} obj={x:100,y:200}

vue浅层检测数据是否修改(直接修改会报错 对象里修改vue无法检测)

关于删除 （首先 鼠标经过对应的小li 会有背景颜色同时也会显示对应的删除按钮）

点击删除按钮 需要拿到id

不要用delete 这些关键词 用handledelete

handleDelete(id){
if(confirm(‘确定删除吗’){

console.log(id)

}

}

弹出询问确定按钮 确定的话才执行if里面语句
